When should I use a class or an ID with CSS?

Choosing to use an id or a class with CSS is linked to the purpose of the CSS you're about to add.

Is the CSS going to be applied multiple times to different parts of your website or of your page? For example, will you be applying a border with particular margins and/or padding to multiple solid blocks of text? Use a class, defined in CSS with a period (.)

Is the CSS structural? For example, are you setting up a grid or flexbox layout and defining where a unique part of the page such as a navigation bar is going to be placed within that structure? Is this the only place on the page you'll be using that CSS? Use an id, defined in CSS with a hashtag (#)

The table below compares the differences between classes and ids.

class id
Can be applied to multiple HTML elements Can only be applied to one HTML element
Multiple classes can be applied to the same HTML element Only one id can be applied to the HTML element
Should be given an adjective as a name eg. bold-text Should be given a noun as a name eg. footer or head-nav
Cannot be used as URL navigation Can be used as the URL in an anchor to navigate to a particular section of a page
A CSS rule applied to a class will override (has specifity over) rules applied to standard html elements A CSS rule applied to an id will override (has specifity over) rules applied to classes as well as to standard html elements

Ids are often used with Javascript. They can also be used as the URL in an anchor to navigate to, then scroll to, a particular section on a webpage. For example, the following URL https://www.colorhexa.com#footer uses the id #footer to take you to the ColorHexa page and scroll down to the footer of that page, using the id of the footer div on that page.
